Basics Overview

In addition to the overall structure of an HTML page (which must be followed!), the most basic HTML tags to encapsulate content are Headings, Paragraphs Links and Images. Each tag has a set of default CSS applied to it, so that is what gives the default look. We will look at these style defaults in the next step, but for now focus on learning about these basic types. 

Headings and Paragraphs (block elements) make up the document body and contained within each of these can be links and images (in-line elements). All HTML tags are generally lowercase, but technically are not case sensitive so would work the same in upper and lower mixed. 

https://www.w3schools.com/html/html_basic.asp

Headings

<h1>This is my document title</h1>

A heading is simple text like a title of a paper or article. There are 6 different levels of headings with default CSS applied to them. You should use Heading 1 first, then a sub-heading within this part of your document would be Heading 2. You can go up to 6-levels deep using the provided heading tags. 

Paragraphs

<p>The quick brown fox jumps over the lazy dog</p>

Paragraphs are blocks of text that will automatically "wrap" based on your screen size. You do not include any "enters" or "line feeds"--just provide a string of text. Paragraphs must not contain other paragraphs. To start a new paragraph, just end the current one using the </p> tag and then start your new one using the <p> tag. Generally, a new paragraph is started on a new line in the code to make it more readable by those reading the source, although that isn't a technical requirement. 

Links

<a href="https://flemingcollege.ca">This will be the text to click</a>

Links (called anchors, why the tag is the letter "A") are the way to point the web page user to another document. Combining multiple pages stored together using links is called a web site. There is a property in the opening tag called href (which is short for http reference) that points to the place you want the reader to go if they click (also called navigate) using this link. Like the P and H# tags before, there is an opening and closing tag with the text you would like the user to click on after the opening tag and before the closing tag. Links should always be within a block level html element, which you now know of two: a heading or paragraph!   

Images

<img src="https://flemingcollege.ca/i/Fleming-Logo-New.svg">

An image can be displayed at any point in a document using an img tag. The src property uses a valid HTTP URL to point at which image should be displayed. It is important that the image size (height and width) be appropriate for your document or it might take a long time to download to your browser, or displace your content in ways you do not want.

Applying styles to HTML elements

The Default Stylesheet

All HTML elements have default CSS elements applied to them. For example, the HTML tag h1 has the following CSS applied by default in Chrome:

h1  {
        display: block;
        font-size: 2em;
        margin-block-start: 0.67em;
        margin-block-end: 0.67em;
        margin-inline-start: 0px;
        margin-inline-end: 0px;
        font-weight: bold;
    }

These style elements make up the default for all elements using h1 and the only way to change the default style is to provide your own CSS to "override" it with your own declarations. The bundle of default declarations for elements is called the User-Agent-Stylesheet and all web browsers provide a similar template, although they can differ slightly (the default above is from Chrome's User-Agent-Stylesheet).

Inline VS Selectors

Inline CSS uses the HTML property style= to directly apply one or more declaration to that element. If the style is compartmentalized into the internal <style> section of a document then there needs to be a way for the web browser to know how to apply those style declarations to the code. For CSS, it uses a selector. You can think of a selector as a search on the page for anything matching what it is. 

The most simple selectors are naming the HTML tag elements themselves. So for some style to apply to all headings, you can use the selector h1. This way anytime the browser sees an HTML tag h1 it will apply that style. 

There are other types of selectors, but for now just focus on using the HTML tags themselves. 

Specificity: CSS Listing Order Matters

A web page applies style in a predictable way. If you style an HTML element to show in blue, then purple which one should be the final colour? The simple rule is last one is the one that will win. Unfortunately it isn't as simple as that, as CSS uses a scoring method called Specificity. It's complex and not important to understand how to calculate this, but to just understand how it works with a few basic selector and CSS types. 

https://www.w3schools.com/css/css_specificity.asp

TL;DR Basically the more specific a style declaration the more important it is. Here is a generalization:

  1. Inline styles are most specific (since they're on the HTML element as a property!)
  2. IDs are like names for elements and (technically) can only occur once, so are next most specific
  3. Classes are the next most specific
  4. Elements are the last in the rules. 

At this point in our lessons we are focusing on inline and inline CSS styles, so knowing that inline style declaration will always override an element from the <style> section of the document is what you need to know.
