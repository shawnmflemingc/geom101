Geom101: Images on Web Pages

Introduction

The web would not be the same if it were not for images being displayed on pages. Here are some key items to understand around images, their placement, and how to best work with them. This documentation is an addendum to the provided documentation at https://www.w3schools.com/html/html_images.asp.

<img src="https://www.w3schools.com/html/pic_trulli.jpg">

The src= must be a valid URL that points to an image. The URL can be relative (for images on the same web server) or fully referenced (for images on other web servers). 

How images are "loaded"

When the initial web page document is being interpreted by your web browser, it is reading line-by-line from the top to the bottom in the document. When it sees a reference to an image it will then spawn a new process to get that image and continue itself interpreting the html document. Therefore it is important for the target space to have a size so the remaining parts of the document can be properly formatted around this yet-to-be-loaded image. That is the purpose of specifying the height and width of the image. 

Specifying the height and width's purpose is to know what the actual size of the image coming in will be. It is not meant to be a method to "resize" the image you are loading. While it is possible to use these html properties to resize an image, it is a bad practice and can significantly impact the page load time or even quality of the image. It is therefore important to specify the height and width of the actual image, not just set the size you want. 

What is my image size?

Take a look at the native size of an image by opening it directly using the url (https://www.w3schools.com/html/pic_trulli.jpg), or if already on the web page in Chrome, right-click and select open image in new tab. Try that on one of the three references to that image here:

Warning The images displayed above are scaled and is a demonstration of what not to do. 

Opening the image directly shows the native (or actual) size of the image in the browser's title. The image above's actual JPG is 500x333 pixels in size, where the width is 500 pixels and the height is 333 pixels. Therefore, the only correct HTML statement with the correct height and width in pixels specified would be:

<img src="https://www.w3schools.com/html/pic_trulli.jpg" height="500" width="333">

Best practices and required for this course

TL;DR The best practice is to specify the actual size of the image to be opened.

While changing the height and width reference does skew or stretch the image on display, it is not the intention of these properties. An image file is compressed and optimized to at the native resolution and showing it any other size is either inefficient in client-server data transmission, can cause poor display quality and causes other issues with accessibility, browser memory usage and performance.

CSS Style for image size

Another method to specify the size of an image is using CSS properties. Some 

<img src="https://www.w3schools.com/html/pic_trulli.jpg" style="width:500px;width:333px;">

Either method is fine, but be consistent. Note that when specifying the height and width in CSS you must specify the units (in the case shown here, px are pixels. 

What if I don't want that to be the size of my image?

You should open the image, change the dimensions to be what you want, then re-save the newly optimized image. Adobe Photoshop, the free open source Paint.net, or the website https://www.photopea.com/ can be used to do this. If the image is to be "scaled" on your page where the size is dependent on the page height or width it is possible to calculate what the maximum image display size should be, then resample the image to that. Then you must set a CSS property of max-width to be the maximum size. 

ALT what?

Required Accessibility standards require all images to include an ALT parameter to describe the image. That would look like this:

<img alt="Trulli - traditional Apulian dry stone hut with a conical roof" src="https://www.w3schools.com/html/pic_trulli.jpg" style="width:500px;width:333px;">

Note that the order of the parameters in an opening HTML tag do not matter, but it is best practice to be consistent in your document. 

Read ALT tag documentation: https://www.w3schools.com/tags/att_img_alt.asp

Image types: PNG, JPG, GIF, SVG!

There are many image formats available to choose from.

This web page has an excellent discussion on this:

https://blog.imagekit.io/jpeg-vs-png-vs-gif-which-image-format-to-use-and-when-c8913ae3e01d?gi=99c48f817bdc

Each has its advantages and disadvantages, but generally speaking here is a short version:

  • Images containing text or logos should be PNG to be most clear
  • Images that have transparent components must be PNG or GIF
  • Images that have require an "Alpha Channel" where some things are translucent must be PNG32. 
  • Pictures of actual things should be JPG (or name it JPEG, just be consistent!)
  • GIF is rarely used, except with mini-animations. 

What is an SVG?

This is a cool VECTOR standard for displaying images on the web! It's not necessary to understand for this course, but a great alternative to having multiple sizes of logos for a single webpage. See https://www.w3schools.com/graphics/svg_intro.asp for more details.
